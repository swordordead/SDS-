# 部署配置说明

## 环境变量一览

| 变量 | 必填 | 说明 |
|------|------|------|
| `GEMINI_API_KEY` | 是（AI 功能） | 兔子 API 密钥，从 [api.tu-zi.com](https://api.tu-zi.com) 获取 |
| `PORT` | 否 | 服务端口，默认 `3100` |
| `NODE_ENV` | 否 | 填 `production` 可关闭 CORS 跨域宽松模式 |

---

## 一、本地开发

```bash
# 1. 复制环境变量文件
cp .env.example .env

# 2. 编辑 .env，填入真实密钥
GEMINI_API_KEY=sk-xxxxxx
PORT=3100
NODE_ENV=development

# 3. 启动（或双击 start.bat）
node server/app.js
```

## 二、Zeabur 部署

1. 将代码推送到 GitHub
2. Zeabur 控制台 → 新建服务 → 选择仓库
3. 配置以下字段：
   - **Build Command**：`cd client && npm install --legacy-peer-deps && npx vite build`
   - **Start Command**：`node server/app.js`
4. 在 **Environment Variables** 面板添加：
   ```
   GEMINI_API_KEY = sk-xxxxxx
   NODE_ENV       = production
   ```
5. 部署完成后访问 Zeabur 分配的域名验证

## 三、公司服务器部署（Linux + PM2）

### 前置要求
- Node.js 18+
- PM2（`npm install -g pm2`）

### 步骤

```bash
# 1. 拉取代码
git clone <仓库地址> /opt/sds-tool
cd /opt/sds-tool

# 2. 安装依赖 & 构建前端
npm install
cd client && npm install --legacy-peer-deps && npx vite build && cd ..

# 3. 配置环境变量（.env 文件不进 git，在服务器手动创建）
cat > .env << EOF
GEMINI_API_KEY=sk-xxxxxx
PORT=3100
NODE_ENV=production
EOF

# 4. 用 PM2 守护进程启动
pm2 start server/app.js --name sds-tool
pm2 save          # 持久化，服务器重启后自动拉起
pm2 startup       # 按提示执行输出的命令，设置开机自启
```

### 常用 PM2 命令

```bash
pm2 status          # 查看运行状态
pm2 logs sds-tool   # 查看实时日志
pm2 restart sds-tool  # 重启服务（更新代码后用）
pm2 stop sds-tool   # 停止服务
```

### Nginx 反向代理（可选，建议配置）

```nginx
server {
    listen 80;
    server_name sds.company.com;   # 替换为公司内网域名或 IP

    location / {
        proxy_pass         http://127.0.0.1:3100;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        # 上传 / 批量生成超时放宽
        proxy_read_timeout 120s;
        client_max_body_size 20m;
    }
}
```

### 更新部署流程

```bash
cd /opt/sds-tool
git pull
cd client && npx vite build && cd ..
pm2 restart sds-tool
```

## API 密钥来源

当前使用 **兔子 API**（`https://api.tu-zi.com`）作为 Gemini 中转：
- 注册地址：https://api.tu-zi.com
- 密钥格式：`sk-xxxxxxxxxxxxxxxx`
- 使用模型：`gemini-2.5-flash-lite-preview-09-2025`

> 如需更换 API 提供商，修改 `server/routes/aiFill.js` 顶部的 `API_URL` 和 `MODEL` 两个常量即可，密钥仍通过 `GEMINI_API_KEY` 环境变量注入。
