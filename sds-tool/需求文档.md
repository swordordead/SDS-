# SDS 自动生成工具 — 需求文档

版本：v0.1.0 | 日期：2026-03-20

---

## 一、项目背景

内部员工需要将 Excel 产品数据快速转换为符合 US SDS（Safety Data Sheet，安全数据表）标准的 PDF / Word 报告，减少手动填写工作量，提高合规文档生产效率。

---

## 二、功能需求

### 2.1 数据输入

1. 支持上传 Excel (.xlsx/.xls) 和 CSV 文件
2. 支持拖拽上传和点击选择
3. 文件大小限制 ≤ 10MB
4. 自动解析第一个 Sheet 的表头和数据行
5. 支持切换 Sheet

### 2.2 字段映射

1. 将 Excel 列名映射到 SDS 标准字段（16 Section，共约 80 个字段）
2. 支持自动映射（基于名称相似度）
3. 映射方案可命名保存/加载（localStorage 持久化）
4. 验证必填字段是否已映射

### 2.3 SDS 编辑

1. 按 16 Section 导航编辑所有字段
2. Section 3 成分表支持动态添加/删除行（化学名称、CAS 号、浓度）
3. 支持 AI 单字段补全（基于产品名和成分）
4. 支持 AI 批量补全所有空字段
5. 实时预览 SDS 排版（iframe 展示 HTML）

### 2.4 文件导出

1. **单份导出**：
   - PDF 格式（Puppeteer 生成，US Letter，1 英寸边距）
   - Word (.docx) 格式（docx 库生成）
   - 支持同时下载两种格式
2. **批量导出**：
   - 勾选多个产品一键生成
   - 打包为 ZIP 文件下载
   - 支持格式选择：PDF / Word / 两种

### 2.5 AI 补全

1. 调用 Gemini API（密钥仅存服务端）
2. 根据产品名称和成分自动生成 H/P 语句、急救措施、消防措施等标准描述
3. AI 结果填入可编辑文本框，用户可修改后确认

## 三、技术架构

### 3.1 单体部署架构

```
Zeabur（单一 Node.js 服务）
┌─────────────────────────────────────────┐
│  Express 服务（端口 3100）               │
│  ├── GET  /*          → Vue 静态文件    │
│  ├── POST /api/upload → Excel 解析      │
│  ├── POST /api/generate → PDF/Word 生成 │
│  ├── POST /api/batch  → 批量 ZIP 导出   │
│  └── POST /api/ai-fill → Gemini AI 补全 │
└─────────────────────────────────────────┘
```

### 3.2 前端流程

```
上传 Excel → 字段映射 → 产品列表 → 编辑 SDS → 下载文件
                                  ↓
                              批量选择 → 批量导出 ZIP
```

## 四、SDS 标准字段（16 Sections）

| Section | 英文标题 | 关键字段 |
|---------|---------|---------|
| 1 | Identification | 产品名称、公司信息、联系电话 |
| 2 | Hazard(s) Identification | GHS 分类、信号词、H/P 语句 |
| 3 | Composition/Ingredients | 成分表（化学名称、CAS 号、浓度）|
| 4 | First-Aid Measures | 吸入、皮肤、眼睛、食入急救 |
| 5 | Fire-Fighting Measures | 灭火剂、消防员防护 |
| 6 | Accidental Release | 泄漏处理、环境防护 |
| 7 | Handling and Storage | 安全操作、储存条件 |
| 8 | Exposure Controls/PPE | 接触限值、个人防护装备 |
| 9 | Physical/Chemical Properties | 外观、pH、闪点、沸点等 17 项 |
| 10 | Stability and Reactivity | 化学稳定性、分解产物 |
| 11 | Toxicological Information | 急性毒性、刺激性、致癌性 |
| 12 | Ecological Information | 生态毒性、降解性 |
| 13 | Disposal Considerations | 废弃处置方法 |
| 14 | Transport Information | UN 编号、运输等级 |
| 15 | Regulatory Information | 美国联邦/州/国际法规 |
| 16 | Other Information | 发布日期、修订版本、免责声明 |

## 五、API 接口规范

### POST /api/upload
- Request: `multipart/form-data`, field `file`
- Response: `{ headers: string[], rows: any[][], sheets: string[], rowCount: number }`

### POST /api/generate
- Request: `{ data: SdsData, format: 'pdf' | 'docx' }`
- Response: Binary file stream（PDF 或 DOCX）

### POST /api/generate/preview
- Request: `{ data: SdsData }`
- Response: HTML text

### POST /api/batch
- Request: `{ products: SdsData[], format: 'pdf' | 'docx' | 'both' }`
- Response: ZIP binary stream

### POST /api/ai-fill
- Request: `{ productName, ingredients, fieldKey, fieldLabel, language? }`
- Response: `{ text: string, fieldKey: string }`

### POST /api/ai-fill/batch
- Request: `{ productName, ingredients, fields: [{key, label}] }`
- Response: `{ results: { [key]: string } }`

## 六、部署说明

### 本地运行
1. `npm install`（根目录）
2. `cd client && npm install --legacy-peer-deps`
3. `cd client && npx vite build`（构建到 server/public/）
4. `node server/app.js`

### Zeabur 部署
1. 推送代码到 GitHub
2. Zeabur 连接仓库，Node.js 服务
3. Build Command: `cd client && npm install --legacy-peer-deps && npx vite build`
4. Start Command: `node server/app.js`
5. 环境变量：`GEMINI_API_KEY`

---

## 七、已知限制

1. Puppeteer PDF 生成依赖 Chromium，首次下载约 200MB
2. 批量生成单次上限 50 个产品（避免超时）
3. AI 功能需要配置 GEMINI_API_KEY 环境变量
4. 文件不持久化，页面刷新数据将丢失（下一版本可加 localStorage）
